第 1 題 L22 L22301 統計學在大數據中的應用
辛普森悖論（Simpson's Paradox）描述的是一種什麼樣的統計現象？
A樣本量越大，P 值越小的現象
B在分組數據中觀察到的趨勢，在將數據合併後消失甚至反轉的現象
C相關性不等於因果關係的現象
D模型過度擬合訓練數據的現象
✅ 正確答案：B
"> 辛普森悖論是指當數據被分成不同的組別進行分析時，在每個組別內都觀察到某種趨勢或關聯，但當把這些組別的數據合併在一起進行分析時，這種趨勢或關聯可能減弱、消失，甚至出現完全相反的結果。這通常是由於存在一個潛在的混淆變數（Lurking Variable），這個變數與分組方式和我們關心的變數都有關聯，導致在合併數據時忽略了這個變數的影響而產生誤導性的結論。
第 2 題 L22 L22301 統計學在大數據中的應用
在對大數據集進行假設檢定時，即使是很小的效應（Effect Size）也可能達到統計顯著性（Statistical Significance, p-value < α）。這現象被稱為什麼問題，我們應該更關注哪個指標？
A多重比較問題（Multiple Comparisons Problem）
BP值駭客（p-hacking）問題，應更關注實際顯著性（Practical Significance）或效應大小
C樣本選擇偏差（Sample Selection Bias）
D中心極限定理（Central Limit Theorem）失效
✅ 正確答案：B
"> 在大數據情境下，由於樣本數 N 非常大，標準誤（Standard Error）會變得非常小。這導致即使效應大小（例如，兩組平均值的差異）非常微小，計算出的檢定統計量也可能很大，從而得到一個極小的 p 值，輕易達到統計顯著。然而，這個微小的效應在實際應用中可能毫無意義。因此，除了看 p 值，更應該關注效應大小（Effect Size，如 Cohen's d, R-squared）或實際顯著性，判斷這個效應在真實世界中是否重要、是否有價值。這個現象本身不是 p-hacking（指操縱數據或分析方法以獲得顯著結果），但容易讓人誤判結果的重要性。

第 4 題 L21 L21301 數據準備與模型選擇
假設你有一個「客戶流失預測」的數據集，其中包含「上次購買日期」這個欄位，為了讓模型能更好地利用這個資訊，可以進行哪種特徵工程？
A直接將日期作為字串輸入模型
B計算「距離現在的天數」（Recency）、提取「星期幾」、「月份」等衍生特徵
C將日期轉換為獨熱編碼
D刪除這個欄位
✅ 正確答案：B
">日期或時間戳欄位通常不能直接作為數值或類別特徵輸入模型。需要進行特徵工程，提取其中有意義的資訊。對於「上次購買日期」，可以計算與當前日期（或數據收集截止日期）的時間差，得到「最近一次購買間隔」（Recency），這通常與客戶流失風險密切相關。此外，還可以提取日期中的周期性成分，如購買發生在星期幾、月份、是否為假日等，這些可能也與客戶行為模式有關。這些衍生出來的新特徵更能被模型所理解和利用。直接使用字串（A）或獨熱編碼（C，會導致維度爆炸）通常效果不佳。刪除欄位（D）則會丟失重要資訊。

第 9 題 L22 L22402 大數據在鑑別式AI應用

A邏輯迴歸 (Logistic Regression)
B支持向量機 (Support Vector Machine, SVM)
C決策樹 (Decision Tree)
D樸素貝氏 (Naive Bayes)
✅ 正確答案：B
">支持向量機（SVM）的基本原理就是最大化間隔（Margin Maximization）。它尋找一個能夠將不同類別數據點分開的決策邊界（超平面），並且使得距離這個邊界最近的數據點（稱為支持向量，Support Vectors）到邊界的距離盡可能大。這個最大化的間隔有助於提高模型的泛化能力。邏輯迴歸是基於 Sigmoid 函數預測機率。決策樹是基於一系列規則進行分裂。Naive Bayes 是基於貝氏定理和獨立性假設。

B可以透過平行化訓練多棵決策樹，並且對特徵和樣本進行抽樣來處理高維度和大數據量
C只能處理線性可分的資料

第 16 題 L21 L21101 自然語言處理技術與應用
機器翻譯 (Machine Translation, MT) 按照技術發展階段，主要經歷了哪些範式？
A僅基於規則的機器翻譯
B僅統計機器翻譯
C僅神經機器翻譯
D基於規則的機器翻譯 (RBMT)、統計機器翻譯 (SMT)、神經機器翻譯 (NMT)
✅ 正確答案：D
">機器翻譯的發展歷程主要可以劃分為三個階段： 基於規則的機器翻譯 (Rule-Based Machine Translation, RBMT): 早期的方法，依賴語言學家手動編寫大量的雙語詞典和語法規則來進行翻譯。優點是語法相對準確，缺點是規則覆蓋有限、構建成本高、難以處理歧義和口語。 統計機器翻譯 (Statistical Machine Translation, SMT): 20 世紀 90 年代興起，利用大規模平行語料庫（成對的原文和譯文句子）學習翻譯模型（如詞對齊、短語翻譯機率）和語言模型。相比 RBMT 效果有顯著提升，但模型複雜，且難以處理長距離依賴。 神經機器翻譯 (Neural Machine Translation, NMT): 2014 年左右開始發展，使用深度神經網路（主要是基於 RNN 的 Encoder-Decoder 架構，後來被 Transformer 取代）直接學習從源語言到目標語言的端到端映射。NMT 在翻譯流暢度和準確度上取得了突破性進展，成為當前主流的機器翻譯範式。
第 17 題 L21 L21104 多模態人工智慧應用
多模態機器翻譯（Multimodal Machine Translation, MMT）相比傳統的純文字機器翻譯，其主要優勢可能來自於？
A大幅降低了翻譯模型的複雜度。
B利用額外的視覺資訊（如圖片）來輔助消除文字中的歧義，提高翻譯質量。
C不再需要大量的平行語料庫進行訓練。
D能夠翻譯圖像本身。
