🧠 科目一: 人工智慧技術應用與規劃

#L21-1
★★★★★
下列何者並未使用人工智慧（AI）或機器學習（ML）技術？
A
使用一組預定義規則來確定最佳移動做法的象棋遊戲
B
使用深度神經網路來提高其準確性的語音識別系統
C
使用感測器和預定義的自動駕駛汽車-定義導航規則
D
使用自然語言處理演算法來理解和反應用戶查詢的聊天機器人
💡 答案解析
正確答案：A

基於規則的系統（Rule-based System）與 AI/ML 的核心差異在於學習能力。

各選項分析：
選項 技術類型 是否使用 AI/ML
A. 預定義規則象棋 規則系統 ❌ 否
B. 深度神經網路語音識別 深度學習（Deep Learning） ✅ 是
C. 自動駕駛汽車 電腦視覺 + ML ✅ 是
D. NLP 聊天機器人 自然語言處理（NLP） ✅ 是

🎯 關鍵區別：
• 規則系統：完全依賴人工編寫的規則，無法從經驗中學習
• AI/ML 系統：能從數據中學習模式，持續改進性能

💡 實例說明：
傳統象棋程式使用 Minimax 演算法和評估函數，所有策略都是程式設計師預先定義的。相比之下，AlphaZero 這類 AI 系統則透過強化學習（Reinforcement Learning）自我對弈，發展出超越人類的策略。
#L21-2
★★★★★
在文本資料處理過程中，通常會需要「將接續的文本轉換為詞彙單位」，以便後續的處理。請問上述所指的是文本資料處理中的哪一個方法？
A
詞形還原（Lemmatization）
B
停用詞移除（Stopword Removal）
C
斷詞（Tokenization）
D
詞頻-逆向文件頻率（Term Frequency-Inverse Document Frequency, TF-IDF）
💡 答案解析
正確答案：C

斷詞（Tokenization）是 NLP 處理流程中的第一步，負責將連續文本切分成有意義的詞彙單位。

📊 NLP 文本處理流程：
步驟 技術名稱 功能說明 範例
1️⃣ 斷詞 將文本切分成詞彙 "我愛AI" → ["我", "愛", "AI"]
2️⃣ 停用詞移除 移除無意義的常見詞 移除 "的"、"是"、"在" 等
3️⃣ 詞形還原 將詞彙轉為基本形式 "running" → "run"
4️⃣ TF-IDF 計算詞彙重要性權重 評估詞彙在文檔中的重要性

🔍 斷詞的類型：
• 詞級斷詞（Word-level）：切分成完整單詞
• 字元級斷詞（Character-level）：切分成單個字元
• 子詞級斷詞（Subword-level）：如 BPE、WordPiece

💡 為什麼斷詞很重要？
機器學習模型無法直接處理原始文本，必須先將文本轉換為數值表示。斷詞是這個轉換過程的基礎，決定了模型能「看到」哪些語言單位。

#L21-4
★★★★★
關於深度學習模型，下列敘述何者不正確？
A
卷積神經網路（Convolutional Neural Networks）適合影像辨識
B
ReLU（Rectified Linear Unit）比 tanh 和 Sigmoid 好，因為 ReLU 可以減緩梯度爆炸與消失的現象
C
遞迴神經網路（Recurrent Neural Networks）適合處理序列相關資料
D
Elman 神經網路（Elman Neural Networks）適合處理影像辨識
💡 答案解析
正確答案：D

Elman 神經網路是一種遞迴神經網路（RNN），主要用於序列資料處理，而非影像辨識。

🧠 深度學習模型與應用領域：
模型類型 適用領域 核心特點 代表架構
CNN ✅ 影像辨識 局部感受野、權重共享 ResNet, VGG, Inception
RNN ✅ 序列資料 記憶前一時刻的狀態 LSTM, GRU, Elman
Transformer ✅ NLP、視覺 自注意力機制 BERT, GPT, ViT
GAN ✅ 生成任務 對抗式訓練 StyleGAN, CycleGAN

⚡ 激活函數比較：
激活函數 公式範圍 優點 缺點
ReLU [0, +∞) ✅ 計算快速
✅ 緩解梯度消失		 ❌ 可能出現神經元死亡
Sigmoid (0, 1) ✅ 輸出可解釋為機率 ❌ 梯度消失嚴重
tanh (-1, 1) ✅ 零中心化 ❌ 梯度消失問題

🎯 Elman 神經網路特點：
• 是最早的 RNN 架構之一（1990年提出）
• 具有上下文層（Context Layer）記憶歷史資訊
• 適用於時間序列預測、語言建模等序列任務
• 現代應用中多被 LSTM 和 GRU 取代

📊 科目二: 大數據處理分析與應用

#L22-1
★★★★
在巨量資料分析班中，共有一年級至四年級，每個年級有 50 個學生，且學生身高呈常態分佈。下列敘述何者不正確？
A
要檢測一年級和二年級的平均身高是否有差異，可以利用 t 檢定
B
要檢測一年級、二年級、三年級之間的平均身高是否有差異，可以利用 t 檢定
C
要檢測二年級、三年級、四年級之間的平均身高是否有差異，可以利用 F 檢定
D
要檢測一年級的平均身高是否等於 170 公分，可以利用卡方檢定
💡 答案解析
正確答案：D

檢測平均值是否等於特定數值應使用 t 檢定，而非卡方檢定（Chi-square Test）。

📊 統計檢定方法比較：
檢定方法 適用情境 比較對象 資料類型
t 檢定 比較 1-2 組平均值 • 單樣本 vs 固定值
• 兩組平均值		 連續型數值
ANOVA
(F 檢定)		 比較 3 組以上平均值 多組平均值 連續型數值
卡方檢定 檢定類別變數關聯性 類別分佈、獨立性 類別型資料

✅ 各選項分析：
選項 A：✅ 正確 - 兩組平均值比較使用獨立樣本 t 檢定
選項 B：❌ 不正確 - 三組以上應使用 ANOVA（F 檢定），而非 t 檢定
選項 C：✅ 正確 - 三組平均值比較使用 ANOVA（F 檢定）
選項 D：❌ 不正確 - 單樣本平均值檢定應使用單樣本 t 檢定

🎯 卡方檢定的正確用途：
• 適合度檢定：檢驗觀察分佈是否符合理論分佈
• 獨立性檢定：檢驗兩個類別變數是否獨立
• 同質性檢定：檢驗不同群體的類別分佈是否相同

💡 範例說明：
假設要檢定「性別」與「是否喜歡數學」的關聯性：
喜歡 不喜歡
男生 30 20
女生 25 25
這種情況才適合使用卡方檢定！
#L22-2
★★★★★
關於接受者操作特徵（ROC）曲線，下列敘述何者正確？
A
ROC 曲線繪製了真陽性率與偽陽性率的關係
B
ROC 曲線用於評估模型的準確性
C
ROC 曲線下的面積（AUC-ROC）始終等於 1
D
ROC 曲線只適用於二元分類問題
💡 答案解析
正確答案：A

ROC（Receiver Operating Characteristic）曲線是評估分類模型性能的重要工具，橫軸為偽陽性率（FPR），縱軸為真陽性率（TPR）。

📈 ROC 曲線核心概念：
指標 英文全名 計算公式 意義
TPR True Positive Rate TP / (TP + FN) 正確識別正例的比例（召回率）
FPR False Positive Rate FP / (FP + TN) 錯誤識別為正例的比例
AUC Area Under Curve ROC 曲線下面積 模型整體性能（0.5-1.0）

🎯 AUC 值的解讀：
• AUC = 1.0：完美分類器
• AUC = 0.9-1.0：優秀
• AUC = 0.8-0.9：良好
• AUC = 0.7-0.8：一般
• AUC = 0.5：隨機猜測（無用）
• AUC < 0.5：比隨機還差

❌ 其他選項為何不正確：
選項 B：ROC 曲線評估的是分類能力，而非單純的準確性（Accuracy）
選項 C：AUC 範圍是 0.5 到 1.0，只有完美分類器才等於 1
選項 D：ROC 曲線可透過 One-vs-Rest 或 One-vs-One 策略擴展到多類別分類

💡 ROC vs PR 曲線：
• ROC 曲線：適合類別平衡的資料集
• PR 曲線（Precision-Recall）：適合類別不平衡的資料集
#L22-3
★★★★
下列何者不屬於特徵工程（Feature Engineering）？
A
轉換（Transformation）
B
萃取（Extraction）
C
挑選（Selection）
D
預測（Prediction）
💡 答案解析
正確答案：D

預測（Prediction）是模型應用階段的任務，而非特徵工程的一部分。

🔧 特徵工程的核心步驟：
步驟 英文名稱 說明 範例
✅ 特徵轉換 Transformation 改變特徵的表示形式 對數轉換、標準化、正規化
✅ 特徵萃取 Extraction 從原始資料中提取新特徵 PCA、文本的 TF-IDF
✅ 特徵選擇 Selection 選擇最相關的特徵子集 過濾法、包裝法、嵌入法
✅ 特徵建構 Construction 組合現有特徵創造新特徵 交互特徵、多項式特徵

🎯 特徵轉換的常見方法：
• 標準化（Standardization）：轉換為均值0、標準差1
• 正規化（Normalization）：縮放到 [0,1] 或 [-1,1]
• 對數轉換：處理偏態分佈
• 編碼：類別變數轉數值（One-Hot、Label Encoding）

🔍 特徵選擇的三大方法：
方法類型 特點 代表技術
過濾法 獨立於模型，計算快速 相關係數、卡方檢定、互資訊
包裝法 依賴模型性能，計算慢 遞迴特徵消除（RFE）
嵌入法 模型訓練中自動選擇 Lasso、決策樹特徵重要性

💡 為什麼特徵工程重要？
業界有句名言：「資料和特徵決定了機器學習的上限，而模型和演算法只是逼近這個上限而已」。好的特徵工程可以：
• 🎯 提升模型準確性
• ⚡ 加快訓練速度
• 🧹 降低過擬合風險
• 💡 增強模型可解釋性
#L22-4
★★★★★
下拉網路商城的老闆擬透過機器學習的方式，利用過往的產品銷售資料，預測下一季的產品銷售數量，以調整現有的庫存水位。下列哪一個類型的模型，比較適合應用在老闆期望的預測目標？
A
決策樹分類器（Decision Tree Classifier）
B
K-means 分群（K-means Clustering）
C
主成份分析（Principal Component Analysis, PCA）
D
線性迴歸（Linear Regression）
💡 答案解析
正確答案：D

預測銷售數量（連續數值）屬於迴歸問題，應使用迴歸模型，而非分類、分群或降維模型。

🎯 機器學習任務類型：
任務類型 目標 輸出類型 代表算法
✅ 迴歸 預測連續數值 數值（如銷售量、價格） 線性迴歸、決策樹迴歸
❌ 分類 預測類別標籤 類別（如是/否、A/B/C） 邏輯迴歸、決策樹分類器
❌ 分群 發現資料群組 群組標籤（無監督） K-means、DBSCAN
❌ 降維 減少特徵數量 低維特徵空間 PCA、t-SNE

📊 各選項分析：
選項 A - 決策樹分類器：
• ❌ 用於分類任務（預測類別）
• 輸出：離散類別標籤
• 範例：預測客戶是否會流失（是/否）

選項 B - K-means 分群：
• ❌ 用於無監督學習（發現資料結構）
• 輸出：群組標籤
• 範例：客戶分群、市場區隔

選項 C - 主成份分析（PCA）：
• ❌ 用於降維（減少特徵數量）
• 輸出：低維度特徵
• 範例：視覺化高維資料、去除冗餘特徵

選項 D - 線性迴歸：
• ✅ 用於迴歸任務（預測連續數值）
• 輸出：連續數值
• 範例：預測銷售量、房價、溫度

🛒 銷售預測的進階模型：
除了線性迴歸，實務上還可考慮：
• 時間序列模型：ARIMA、Prophet
• 樹模型：Random Forest Regressor、XGBoost
• 深度學習：LSTM、Transformer

💡 實務建議：
銷售預測通常需要考慮：
• 📅 季節性：節慶、促銷活動
• 📈 趨勢：長期增長或衰退
• 🔄 週期性：週末、月底效應
• 🎯 外部因素：經濟指標、競爭對手

★★★★
對於低結構化的文本或圖像資料，下列哪一種特徵工程（Feature Engineering）方法最為適用？
A
特徵改善（Feature Improvement）
B
特徵建構（Feature Construction）
C
特徵學習（Feature Learning）
D
特徵選擇（Feature Selection）
💡 答案解析
正確答案：C

對於低結構化資料（如文本、圖像、音訊），特徵學習（Feature Learning）能自動學習有效的特徵表示，無需人工設計。

📊 資料結構化程度與特徵工程：
資料類型 結構化程度 適合方法 範例
結構化資料 高 特徵選擇、特徵建構 表格資料、數值資料
半結構化資料 中 特徵萃取 + 特徵學習 JSON、XML、日誌
非結構化資料 低 ✅ 特徵學習 文本、圖像、音訊、影片

🧠 特徵學習的核心技術：
資料類型 特徵學習方法 代表模型
📝 文本 詞嵌入、語言模型 Word2Vec、BERT、GPT

在 MapReduce 計算框架中，關於 Map 和 Reduce 所負責處理資料的問題，下列敘述何者正確？
A
Map：一組資料映射成另一組資料；Reduce：統合與歸納資料
B
Map：地圖式的搜索資料；Reduce：統合與歸納資料
C
Map：一組資料映射成另一組資料；Reduce：過濾不符合的資料
D
Map：一組資料映射成另一組資料；Reduce：生成更多的資料
💡 答案解析
正確答案：A

MapReduce 是分散式運算框架的經典模型，由 Google 提出，用於處理大規模資料集。

🔄 MapReduce 運作流程：
階段 功能 輸入 輸出
1️⃣ Map 將資料映射轉換 原始資料 鍵值對 (key, value)
2️⃣ Shuffle 依 key 分組排序 Map 輸出 分組後的鍵值對
3️⃣ Reduce 統合歸納資料 分組後的資料 最終結果

📝 經典範例：字數統計（Word Count）
階段 處理內容
輸入 "Hello World Hello AI"
Map 階段 (Hello, 1), (World, 1), (Hello, 1), (AI, 1)
Shuffle 階段 (Hello, [1, 1]), (World, [1]), (AI, [1])
Reduce 階段 (Hello, 2), (World, 1), (AI, 1)

🎯 Map 和 Reduce 的核心功能：
Map 階段：
• ✅ 映射轉換：將輸入資料轉換為鍵值對
• ✅ 並行處理：多個 Map 任務同時執行
• ✅ 資料分散：將大資料集分割處理

Reduce 階段：
• ✅ 統合歸納：將相同 key 的 value 聚合
• ✅ 彙總計算：執行求和、計數、平均等操作
• ✅ 結果輸出：產生最終結果

💡 MapReduce 的應用場景：
• 📊 資料統計：字數統計、日誌分析
• 🔍 搜尋引擎：建立倒排索引
• 📈 資料探勘：關聯規則挖掘
• 🧮 機器學習：分散式訓練

🚀 MapReduce 的優勢：
• ⚡ 可擴展性：輕鬆處理 PB 級資料
• 🛡️ 容錯性：自動處理節點故障
• 🔧 簡單性：開發者只需實作 Map 和 Reduce 函數
• 🌐 分散式：充分利用叢集資源

🔧 相關技術生態：
• Hadoop：開源 MapReduce 實作
• Spark：記憶體運算，比 MapReduce 快 10-100 倍
• Hive：基於 MapReduce 的 SQL 查詢引擎

★★★★
下列何種卷積神經網路（Convolutional Neural Networks, CNN）是將卷積層加寬而非加深？
A
R-CNN
B
Inception
C
ResNet
D
VGG19
💡 答案解析
正確答案：B

Inception 網路（又稱 GoogLeNet）的核心創新是加寬網路而非加深，透過 Inception Module 在同一層並行使用多種尺寸的卷積核。

🏗️ CNN 架構設計策略：
網路架構 設計策略 核心特點 層數
✅ Inception 加寬 並行多尺度卷積 22 層
VGG19 加深 堆疊 3×3 卷積 19 層
ResNet 加深 + 殘差連接 跳躍連接解決梯度消失 50-152 層
R-CNN 物件偵測 區域提議 + CNN -

🔍 Inception Module 結構：
並行分支 卷積核大小 作用
分支 1 1×1 卷積 捕捉點特徵、降維
分支 2 1×1 → 3×3 卷積 捕捉中等尺度特徵
分支 3 1×1 → 5×5 卷積 捕捉大尺度特徵
分支 4 3×3 池化 → 1×1 卷積 保留空間資訊
最後將四個分支的輸出串接（Concatenate）在一起。

🎯 為什麼要「加寬」？
• 🔍 多尺度特徵：同時捕捉不同大小的物件
• 💡 增加表達能力：不增加深度也能提升性能
• ⚡ 計算效率：1×1 卷積降維減少計算量
• 🎨 特徵豐富性：多種卷積核提供多樣化特徵

📊 各架構的演進：
年份 架構 ImageNet Top-5 錯誤率 創新點
2012 AlexNet 16.4% 首個深度 CNN
2014 VGG 7.3% 小卷積核堆疊
2014 Inception 6.7% ✅ 多尺度並行
2015 ResNet 3.6% 殘差連接

💡 Inception 的版本演進：
• Inception v1（GoogLeNet）：基礎版本
• Inception v2/v3：加入批次正規化、分解卷積
• Inception v4：結合 ResNet 的殘差連接
• Inception-ResNet：混合架構

🚀 實務應用：
Inception 架構特別適合：
• 🖼️ 需要處理多尺度物件的影像辨識
• ⚡ 對計算資源有限制的場景
• 🎯 需要平衡準確性和效率的應用

當模型的訓練誤差（Training Error）低、但測試誤差（Test Error）很大時，這通常是在訓練過程中產生下列哪一種情況？
A
模型的泛化能力強
B
模型出現過度擬合（Overfitting）
C
模型出現欠擬合（Underfitting）
D
訓練資料和測試資料之間沒有相關性
💡 答案解析
正確答案：B

訓練誤差低、測試誤差高是過度擬合（Overfitting）的典型特徵，表示模型記住了訓練資料的細節和雜訊，但無法泛化到新資料。

📊 過度擬合 vs 欠擬合：
情況 訓練誤差 測試誤差 原因 解決方法
✅ 過度擬合 ❌ 很低 ❌ 很高 模型太複雜 正規化、Dropout、更多資料
欠擬合 ❌ 很高 ❌ 很高 模型太簡單 增加模型複雜度、更多特徵
理想狀態 ✅ 低 ✅ 低 模型適中 -

🎯 過度擬合的成因：
• 🧠 模型過於複雜：參數太多，學習能力太強
• 📉 訓練資料太少：無法代表真實分佈
• ⏰ 訓練時間過長：過度學習訓練集的雜訊
• 🎲 缺乏正規化：沒有約束模型複雜度

🛡️ 防止過度擬合的方法：
方法 原理 適用場景
正規化 L1/L2 懲罰大權重 線性模型、神經網路
Dropout 隨機丟棄神經元 深度神經網路
早停法 監控驗證誤差提前停止 所有迭代訓練
資料增強 增加訓練資料多樣性 影像、文本任務
集成學習 組合多個模型 所有模型類型

📈 訓練曲線診斷：
曲線特徵 診斷結果 建議
訓練誤差↓ 測試誤差↓ ✅ 正常學習 繼續訓練
訓練誤差↓ 測試誤差↑ ❌ 過度擬合 停止訓練、加正規化
訓練誤差高且平穩 ❌ 欠擬合 增加模型複雜度

💡 實務建議：
• 📊 使用驗證集監控過度擬合
• 🔍 觀察學習曲線及早發現問題
• ⚖️ 在模型複雜度和泛化能力間取得平衡
• 🎯 優先考慮增加訓練資料

★★★★
下列哪一種指標通常用於評估迴歸模型的性能？
A
R²
B
F1-分數
C
曲線下面積（AUC）
D
Precision
💡 答案解析
正確答案：A

R²（R-squared，決定係數）是評估迴歸模型性能的核心指標，表示模型能解釋的變異比例。

📊 迴歸模型評估指標：
指標 全名 公式 範圍 越好
✅ R² R-squared 1 - (SSres / SStot) (-∞, 1] 越接近 1
MAE Mean Absolute Error Σ|y - ŷ| / n [0, +∞) 越小
MSE Mean Squared Error Σ(y - ŷ)² / n [0, +∞) 越小
RMSE Root Mean Squared Error √MSE [0, +∞) 越小

🎯 R² 的解讀：
• R² = 1.0：完美預測，所有變異都被解釋
• R² = 0.8-0.9：優秀，大部分變異被解釋
• R² = 0.5-0.8：良好，一半以上變異被解釋
• R² = 0.0：模型等同於預測平均值
• R² < 0：模型比預測平均值還差

❌ 分類模型評估指標（不適用於迴歸）：
指標 用途 適用任務
F1-分數 精確率和召回率的調和平均 ❌ 分類任務
AUC ROC 曲線下面積 ❌ 分類任務
Precision 預測為正例中真正為正例的比例 ❌ 分類任務
Recall 實際正例中被正確預測的比例 ❌ 分類任務

🔍 各指標的特點比較：
MAE（平均絕對誤差）：
• ✅ 容易理解，單位與目標變數相同
• ✅ 對離群值不敏感
• ❌ 不可微分（優化較困難）

MSE（均方誤差）：
• ✅ 可微分，適合梯度下降
• ✅ 懲罰大誤差
• ❌ 對離群值敏感
• ❌ 單位是目標變數的平方

RMSE（均方根誤差）：
• ✅ 單位與目標變數相同
• ✅ 懲罰大誤差
• ❌ 對離群值敏感

R²（決定係數）：
• ✅ 標準化指標，易於比較不同模型
• ✅ 直觀表示解釋能力
• ❌ 可能被特徵數量影響（使用調整後 R² 改善）

💡 實務選擇建議：
• 🎯 一般情況：使用 R² 和 RMSE 組合
• 📊 有離群值：優先使用 MAE
• 🔬 科學研究：報告 R²、RMSE、MAE
• 💼 業務溝通：使用 MAE（最直觀）

近年來，深度學習研究與應用蓬勃發展，但數據本身可能存在什麼潛在問題？
A
數據標註品質鮮少被討論，但它卻直接影響模型性能
B
數據品質是完美可信賴的
C
大部分情況下，數據不存在類別不平衡問題
D
數據不需要領域知識的輔助
💡 答案解析
正確答案：A

數據標註品質是深度學習中最容易被忽視但影響最大的因素之一。業界有句名言：「Garbage In, Garbage Out」（垃圾進，垃圾出）。

🎯 數據標註的常見問題：
問題類型 說明 影響 解決方案
標註錯誤 人為失誤導致標籤錯誤 模型學到錯誤模式 多人標註、交叉驗證
標註不一致 不同標註者標準不同 模型混淆 制定明確標註規範
標註偏見 標註者主觀偏好 模型產生偏見 多樣化標註團隊
標註不完整 遺漏重要資訊 模型性能受限 詳細標註指南

📊 數據品質的其他常見問題：
1️⃣ 類別不平衡：
• 問題：某些類別樣本數遠多於其他類別
• 影響：模型偏向多數類別
• 解決：過採樣、欠採樣、SMOTE、類別權重調整

2️⃣ 數據偏見：
• 問題：訓練數據不能代表真實世界分佈
• 影響：模型在特定群體表現差
• 解決：多樣化數據收集、公平性評估

3️⃣ 數據洩漏：
• 問題：測試資訊洩漏到訓練集
• 影響：過度樂觀的性能評估
• 解決：嚴格分離訓練/測試集、時間序列切分

4️⃣ 缺失值：
• 問題：資料不完整
• 影響：模型無法處理或性能下降
• 解決：填補、刪除、使用能處理缺失值的模型

💡 提升數據標註品質的方法：
方法 說明 成本
多人標註 同一資料由多人標註，取共識 高
專家審核 領域專家檢查標註結果 高
標註指南 制定詳細的標註規範和範例 中
標註工具 使用輔助工具提高效率和一致性 中
主動學習 優先標註對模型最有價值的樣本 低

🔍 為什麼領域知識很重要？
• 🏥 醫療影像：需要醫生標註病灶
• ⚖️ 法律文本：需要律師理解法律條文
• 🏭 工業檢測：需要工程師識別瑕疵
• 🌐 多語言 NLP：需要母語者標註

📈 數據品質對模型性能的影響：
研究顯示：
• 標註錯誤率 5% 可能導致模型準確率下降 10-20%
• 提升數據品質的投資報酬率通常高於改進模型架構
• 高品質小數據集往往優於低品質大數據集

🎯 實務建議：
• 📊 定期審計數據品質
• 🔄 建立標註品質回饋機制
• 💰 在數據標註上投入足夠資源
• 🤝 與領域專家密切合作
• 📝 記錄標註過程和決策
