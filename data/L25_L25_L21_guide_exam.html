📝 題目 #1
⭐⭐⭐⭐⭐
下列何者為自然語言處理（NLP, Natural Language Processing）中的詞嵌入技術（Word Embedding），能將文字轉換為向量（Vector）以利機器學習處理？
A
TF-IDF
B
Word2Vec
C
停用詞（Stop Words）
D
詞袋模型（Bag-of-Words, BoW）
💡 詳細解析

正確答案：B

詞嵌入技術是將文字轉換成稠密向量（Dense Vector）的方法，讓電腦能理解詞彙之間的語義關係。

技術名稱 類型 特性說明
Word2Vec ✅ 詞嵌入 將詞彙轉換為低維度稠密向量，能捕捉語義關係（如：國王-男人+女人≈皇后）
TF-IDF ❌ 統計方法 計算詞彙重要性的統計指標，產生稀疏向量（Sparse Vector）
Stop Words ❌ 前處理 常見無意義詞彙（如：的、是、在），用於文本清理
BoW ❌ 表示方法 統計詞彙出現次數，忽略詞序，產生高維稀疏向量

🎯 考試重點：

  • Word2Vec 包含兩種訓練模型：CBOW（Continuous Bag of Words）和 Skip-gram
  • 詞嵌入能捕捉語義相似性和語法關係
  • 其他常見詞嵌入技術：GloVe、FastText、BERT 的上下文嵌入

📝 題目 #3
⭐⭐⭐⭐⭐
語言模型 BERT（Bidirectional Encoder Representations from Transformers）的主要特性為何？
A
使用遞迴神經網路（RNN, Recurrent Neural Network）
B
無法處理上下文資訊
C
為單向語言模型（Unidirectional Language Model）
D
具雙向上下文理解能力（Bidirectional Context Understanding）
💡 詳細解析

正確答案：D

BERT 是 Google 在 2018 年提出的革命性語言模型，其核心特色是雙向編碼，能同時考慮詞彙的左右上下文。

模型特性 說明 優勢
🔄 雙向編碼 同時讀取前後文脈絡 更準確理解詞義（如：「銀行」在不同語境的意思）
🏗️ Transformer 架構 使用自注意力機制（Self-Attention） 平行處理，訓練速度快
🎭 遮罩語言模型 MLM（Masked Language Model） 隨機遮蔽詞彙進行預測訓練
🔗 下一句預測 NSP（Next Sentence Prediction） 理解句子間的關係

🎯 與其他模型比較：

  • GPT：單向（從左到右）生成式模型
  • ELMo：雙向但分別訓練，非真正雙向
  • BERT：真正的雙向同時編碼

💼 應用場景：問答系統、文本分類、NER、情感分析、語義相似度計算

📝 題目 #6
⭐⭐⭐⭐⭐
生成對抗網路（GAN, Generative Adversarial Network）的訓練包含哪兩個核心組件？
A
分類器與分群器
B
產生器（Generator）與鑑別器（Discriminator）
C
強化器與回饋器
D
解碼器（Decoder）與編碼器（Encoder）
💡 詳細解析

正確答案：B

GAN 由 Ian Goodfellow 於 2014 年提出，透過對抗訓練機制生成逼真的資料。

組件 角色 目標 比喻
產生器
（Generator, G）		 🎨 創作者 生成逼真的假資料
欺騙鑑別器		 偽鈔製造者
鑑別器
（Discriminator, D）		 🔍 評判者 區分真實與生成資料
不被產生器欺騙		 警察鑑定專家

🔄 訓練過程（對抗機制）：

  1. 產生器從隨機雜訊（Random Noise）生成假資料
  2. 鑑別器學習區分真實資料和假資料
  3. 產生器根據鑑別器的反饋改進生成品質
  4. 兩者互相競爭、共同進步，直到達到納許均衡（Nash Equilibrium）

🎯 GAN 變體：

  • DCGAN：使用CNN架構，生成高品質影像
  • CGAN（Conditional GAN）：條件式生成，可控制生成內容
  • StyleGAN：生成超逼真人臉
  • CycleGAN：影像風格轉換（如照片→畫作）
  • Pix2Pix：成對影像轉換

💼 應用領域：影像生成、風格轉換、超解析度、資料增強、藝術創作

📝 題目 #8

多模態人工智慧（Multimodal AI）的核心目標為何？
A
提升硬體運算效能
B
同時處理並整合來自不同感知類型（Modality）的資料

專注於純文字輸入的分析
D

多模態 AI 模仿人類的多感官整合能力，能同時理解和處理不同類型的資料（文字、影像、聲音、影片等）。

模態類型 資料形式 範例
📝 文字模態 自然語言文本 文章、對話、標題
🖼️ 視覺模態 影像、影片 照片、圖表、動畫
🔊 聲音模態 音訊、語音 對話、音樂、環境音
📊 數值模態 結構化資料 感測器數據、時間序列

  • 🔗 跨模態理解：理解不同模態間的關聯（如圖片與描述文字的對應）
  • 🎨 跨模態生成：從一種模態生成另一種（如文字生成圖片）
  • 🧠 更全面的理解：結合多種資訊源，做出更準確的判斷
  • 🌐 更自然的互動：更接近人類的溝通方式

💡 經典多模態模型：

  • CLIP：影像-文字對齊
  • DALL-E：文字生成影像
  • Flamingo：視覺語言模型
  • GPT-4V：支援視覺輸入的 GPT-4
  • Gemini：Google 的原生多模態模型

💼 實際應用：

  • 🤖 智慧助理：理解語音+視覺+文字
  • 🏥 醫療診斷：整合影像+病歷+檢驗數據
  • 🚗 自動駕駛：融合攝影機+雷達+地圖
  • 🎬 影片理解：分析畫面+聲音+字幕
📝 題目 #9
⭐⭐⭐⭐⭐
CLIP（Contrastive Language-Image Pre-training）模型結合了哪兩種模態進行訓練？
A
聲音與影像
B
影像與文字
C
文字與數值
D
圖形與表格
💡 詳細解析

正確答案：B

CLIP 是 OpenAI 於 2021 年發表的視覺-語言模型，透過對比學習（Contrastive Learning）將影像和文字映射到同一個嵌入空間（Embedding Space）。

特性 說明 優勢
🎯 對比學習 學習匹配的圖文對相似度高
不匹配的相似度低		 無需大量標註資料
📚 大規模訓練 使用 4 億組圖文對訓練 強大的泛化能力
🔄 零樣本學習 Zero-shot Learning
無需微調即可分類		 適應新類別
🌐 跨模態檢索 文字搜尋圖片
圖片搜尋文字		 靈活的應用方式

🔧 CLIP 架構：

  • 影像編碼器：使用 Vision Transformer 或 ResNet
  • 文字編碼器：使用 Transformer
  • 對比損失：最大化正確配對的相似度，最小化錯誤配對

💼 CLIP 的應用：

  • 🔍 以圖搜圖：用文字描述搜尋相似圖片
  • 🏷️ 零樣本分類：不需訓練即可分類新類別
  • 🎨 文字生成圖片：作為 DALL-E、Stable Diffusion 的基礎
  • 🤖 視覺問答：理解圖片內容並回答問題

🌟 後續發展：

  • ALIGN（Google）：類似概念，更大規模
  • BLIP：改進的視覺-語言預訓練
  • LLaVA：結合大型語言模型的視覺理解

在特徵工程（Feature Engineering）中，進行特徵標準化（如 Z-score 標準化）的主要目的是？

D

特徵標準化將不同尺度的特徵轉換到相同範圍，避免數值大的特徵主導模型學習。

🎯 為什麼需要標準化？

  • 避免特徵主導：
    • 範例：年收入（0-1000萬）vs. 年齡（0-100）
    • 未標準化時，收入會主導模型學習
    • 標準化後，兩者權重相當
  • 加速收斂：
    • 梯度下降法在標準化後收斂更快
    • 減少訓練時間
  • 提升效能：
    • 距離相關演算法（KNN、K-means）需要標準化
    • 正則化方法（L1、L2）效果更好

  • 保存參數：部署時需要保存標準化參數（均值、標準差等）
  • 類別特徵：不要標準化類別型特徵（如性別、城市）

🎯 缺失值類型：

  • MCAR（Missing Completely At Random）：
    • 缺失完全隨機，與任何變數無關
    • 範例：感測器隨機故障
    • 處理：任何方法都可以
  • MAR（Missing At Random）：
    • 缺失與觀察到的變數有關
    • 範例：年輕人較不願透露收入
    • 處理：使用模型插補
  • MNAR（Missing Not At Random）：
    • 缺失與未觀察到的值本身有關
    • 範例：高收入者不願透露收入
    • 處理：最困難，需領域知識

💡 插補策略選擇：

  • 缺失率 < 5%：可考慮刪除
  • 缺失率 5-20%：使用統計或模型插補
  • 缺失率 > 20%：考慮刪除該特徵或使用進階方法
  • 數值型特徵：平均值、中位數、KNN
  • 類別型特徵：眾數、新增「缺失」類別
  • 時間序列：前後值填補、線性插值

💻 Python 實作範例：

from sklearn.impute import SimpleImputer, KNNImputer

# 平均值插補
imputer = SimpleImputer(strategy='mean')
X_imputed = imputer.fit_transform(X)

# 中位數插補
imputer = SimpleImputer(strategy='median')

# 眾數插補
imputer = SimpleImputer(strategy='most_frequent')

# KNN 插補
imputer = KNNImputer(n_neighbors=5)
X_imputed = imputer.fit_transform(X)

⚠️ 注意事項：

  • ✅ 在訓練集上 fit，在測試集上 transform
  • ✅ 記錄插補策略，部署時使用相同方法
  • ✅ 考慮新增「是否缺失」的指示變數
  • ✅ 分析缺失模式，理解缺失原因
  • ❌ 不要在整個資料集上插補（會造成資料洩漏）
