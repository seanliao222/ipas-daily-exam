📝 題目 #1
⭐⭐⭐⭐⭐
若欲描述隨機變數（Random Variable）在連續範圍內取值的機率分佈（Probability Distribution），最適合使用下列哪一種函數？
A
機率質量函數（PMF, Probability Mass Function）
B
條件機率（Conditional Probability）
C
機率密度函數（PDF, Probability Density Function）
D
累積分佈函數（CDF, Cumulative Distribution Function）
💡 詳細解析

正確答案：C

在機器學習中，理解連續型隨機變數與離散型隨機變數的差異非常重要。

函數類型 適用變數 特性 範例
✅ PDF
機率密度函數		 連續型 曲線下面積代表機率
單點機率為 0		 身高、體重、溫度
PMF
機率質量函數		 離散型 直接給出每個值的機率 骰子點數、人數
CDF
累積分佈函數		 兩者皆可 P(X ≤ x) 的累積機率 通用於所有分佈

🎯 PDF 的重要特性：

  • 非負性：f(x) ≥ 0 對所有 x 成立
  • 總面積為 1：∫ f(x)dx = 1（整個範圍積分）
  • 區間機率：P(a ≤ X ≤ b) = ∫[a,b] f(x)dx
  • 單點機率為 0：P(X = x) = 0（連續型特性）

📊 常見的連續型分佈：

  • 常態分佈（Normal Distribution）：最常用，鐘形曲線
  • 均勻分佈（Uniform Distribution）：所有值機率相等
  • 指數分佈（Exponential Distribution）：等待時間模型
  • 貝塔分佈（Beta Distribution）：機率的機率分佈

💼 機器學習應用：

  • 特徵分佈分析：了解資料特性
  • 異常值偵測：低機率密度區域
  • 生成模型：VAE、GAN 學習資料分佈
  • 貝氏推論：先驗與後驗分佈
📝 題目 #2
⭐⭐⭐⭐⭐
若一個隨機變數服從伯努利分佈（Bernoulli Distribution），則該變數可能取值為下列哪一組？
A
任何實數
B
0 或 1
C
正整數
D
任意整數
💡 詳細解析

正確答案：B

伯努利分佈是最簡單的離散型機率分佈，用於描述單次二元實驗的結果（成功或失敗）。

特性 說明 數學表示
取值範圍 只有兩個可能值 X ∈ {0, 1}
參數 成功機率 p 0 ≤ p ≤ 1
機率質量函數 P(X=1) = p
P(X=0) = 1-p		 P(X=k) = p^k(1-p)^(1-k)
期望值 平均值 E[X] = p
變異數 離散程度 Var(X) = p(1-p)

🎯 伯努利分佈的應用場景：

  • 🎲 單次實驗：拋一次硬幣（正面/反面）
  • ✅ 二元分類：郵件是否為垃圾郵件
  • 🏥 醫療診斷：病人是否患病
  • 🔍 點擊預測：用戶是否點擊廣告
  • ⚠️ 異常偵測：交易是否異常

📊 相關分佈：

  • 二項分佈（Binomial Distribution）：n 次獨立伯努利試驗的成功次數
  • 幾何分佈（Geometric Distribution）：首次成功所需的試驗次數
  • 負二項分佈（Negative Binomial Distribution）：達到 r 次成功所需的試驗次數

💼 機器學習中的應用：

  • 邏輯迴歸（Logistic Regression）：假設目標變數服從伯努利分佈
  • 貝氏分類器（Naive Bayes Classifier）：二元特徵建模
  • GAN 的鑑別器：真/假二元判斷
  • 強化學習：動作成功/失敗的建模

🔑 記憶口訣：伯努利 = 單次 + 二元（0或1）

📝 題目 #3
⭐⭐⭐⭐⭐
在主成分分析（PCA, Principal Component Analysis）中，常使用哪一種矩陣分解（Matrix Decomposition）技術來找出資料的主軸方向？
A
矩陣求逆（Matrix Inversion）
B
矩陣轉置（Matrix Transpose）
C
特徵值分解（Eigenvalue Decomposition）
D
條件機率分解（Conditional Probability Decomposition）
💡 詳細解析

正確答案：C

PCA 是最常用的降維技術，透過特徵值分解找出資料變異最大的方向。

步驟 操作 目的
1️⃣ 資料標準化 中心化（減去平均值）
可選：標準化（除以標準差）		 消除量綱影響
2️⃣ 計算共變異數矩陣 Σ = (1/n)X^T X 衡量特徵間的相關性
3️⃣ 特徵值分解 Σ = VΛV^T
V: 特徵向量
Λ: 特徵值		 找出主成分方向
4️⃣ 選擇主成分 選取前 k 個最大特徵值
對應的特徵向量		 保留最重要的資訊
5️⃣ 資料轉換 Y = XV_k 投影到低維空間

🎯 特徵值與特徵向量的意義：

  • 特徵向量（Eigenvector）：主成分的方向（資料變異最大的軸）
  • 特徵值（Eigenvalue）：該方向上的變異量（重要性）
  • 排序：特徵值由大到小排列，對應最重要到最不重要的方向
  • 解釋變異比例：λ_i / Σλ_j 表示第 i 個主成分解釋的變異比例

📊 如何選擇主成分數量 k？

  • 累積解釋變異：選擇能解釋 85-95% 變異的前 k 個主成分
  • 陡坡圖法：觀察特徵值的碎石圖（Scree Plot），找到「手肘點」
  • Kaiser 準則：保留特徵值 > 1 的主成分（標準化後）
  • 交叉驗證：根據下游任務的效能決定

💼 PCA 的應用：

  • 🎨 資料視覺化：降至 2D/3D 進行繪圖
  • ⚡ 加速訓練：減少特徵數量，降低計算成本
  • 🔍 特徵工程：去除冗餘特徵，減少共線性
  • 🖼️ 影像壓縮：保留主要資訊，減少儲存空間
  • 🔊 降噪：去除低變異的雜訊成分

⚠️ PCA 的限制：

  • 假設線性關係，無法捕捉非線性結構
  • 對離群值敏感，建議先處理異常值
  • 主成分難以解釋，是原始特徵的線性組合
  • 需要標準化，否則大數值特徵會主導結果

🔄 相關技術：

  • SVD（Singular Value Decomposition）：更穩定的 PCA 實作方式
  • 核主成分分析（Kernel PCA）：處理非線性關係
  • t-SNE：非線性降維，適合視覺化
  • UMAP：更快的非線性降維方法

在梯度下降（Gradient Descent）中，「梯度（Gradient）」代表下列哪一項？
A
資料點個數
B
模型精確度
C
損失函數（Loss Function）對參數的偏微分（Partial Derivative）
D
模型計算速度
💡 詳細解析

正確答案：C

梯度是損失函數對所有參數的偏微分向量，指出損失函數增長最快的方向。

概念數學表示意義
梯度∇L(θ) = [∂L/∂θ₁, ∂L/∂θ₂, ...]損失增長最快的方向
梯度下降θ ← θ - α∇L(θ)沿負梯度方向更新參數

  • SGD（Stochastic Gradient Descent）：每次用一筆資料，快但不穩定
  • Mini-batch GD：折衷方案，使用小批次資料
📝 題目 #6
⭐⭐⭐⭐⭐
在進行假設檢定（Hypothesis Testing）時，顯著水準（Significance Level, α）表示什麼？
A
平均值的大小
B
模型運算速度
C
拒絕虛無假設（Null Hypothesis）時所容許的型一錯誤（Type I Error）機率
D
資料筆數多少
💡 詳細解析

正確答案：C

顯著水準 α 是我們願意承擔錯誤拒絕真實虛無假設的最大機率，常見值為 0.05 或 0.01。

錯誤類型實際情況決策後果
✅ 正確H₀ 為真接受 H₀正確決策
❌ 型一錯誤
（α）		H₀ 為真拒絕 H₀誤報（False Positive）
❌ 型二錯誤
（β）		H₀ 為假接受 H₀漏報（False Negative）
✅ 正確H₀ 為假拒絕 H₀正確決策（檢定力 1-β）

🎯 假設檢定步驟：

  • 1️⃣ 建立假設：H₀（虛無假設）vs H₁（對立假設）
  • 2️⃣ 選擇顯著水準：通常 α = 0.05
  • 3️⃣ 計算檢定統計量：如 t 值、z 值
  • 4️⃣ 計算 p 值：觀察到此結果的機率
  • 5️⃣ 做出決策：若 p < α，拒絕 H₀
📝 題目 #7
⭐⭐⭐⭐⭐
若資料集中存在極端值（Extreme Value），以下哪一種集中趨勢指標（Central Tendency Measure）較不受影響？
A
平均數（Mean）
B
變異數（Variance）
C
標準差（Standard Deviation）
D
中位數（Median）
💡 詳細解析

正確答案：D

中位數是穩健統計量（Robust Statistic），不受極端值影響，因為它只關注資料的中間位置。

統計量定義對極端值敏感度適用情境
平均數所有值的總和 ÷ 個數❌ 非常敏感資料對稱、無極端值
✅ 中位數排序後的中間值✅ 不敏感有極端值或偏態分佈
眾數出現最多次的值✅ 不敏感類別資料
變異數離均差平方的平均❌ 非常敏感衡量離散程度
標準差變異數的平方根❌ 非常敏感與原始資料同單位

📊 範例說明：

資料：[1, 2, 3, 4, 100]

  • 平均數：(1+2+3+4+100)/5 = 22（被 100 嚴重拉高）
  • 中位數：3（不受 100 影響）

🎯 穩健統計量：

  • 中位數（Median）
  • IQR（Interquartile Range, 四分位距）
  • MAD（Median Absolute Deviation, 中位數絕對偏差）

邏輯迴歸（Logistic Regression）模型在處理二元分類問題（Binary Classification）時，通常假設目標變數服從哪種分佈？
A

B

C
伯努利分佈（Bernoulli Distribution）
D
泊松分佈（Poisson Distribution）
💡 詳細解析

正確答案：C

邏輯迴歸是廣義線性模型（GLM, Generalized Linear Model）的一種，假設目標變數 Y 服從伯努利分佈（Y ∈ {0, 1}）。

模型目標變數分佈連結函數應用
線性迴歸常態分佈恆等函數預測連續值

🔧 邏輯迴歸的核心：

  • 線性組合：z = w₁x₁ + w₂x₂ + ... + b

  • 輸出機率：P(Y=1|X) = σ(z)，範圍 [0, 1]
  • 決策邊界：當 P(Y=1|X) ≥ 0.5 時，預測為類別 1

📊 損失函數：

交叉熵損失（Cross-Entropy Loss）：
L = -[y log(ŷ) + (1-y) log(1-ŷ)]

💼 應用場景：垃圾郵件偵測、疾病診斷、信用評分、點擊率預測

正確答案：A

🎯 混淆矩陣與公平性：

預測為正TP（真陽性）FP（假陽性）

  • TPR（召回率）：TP / (TP + FN)
  • FPR（假陽性率）：FP / (FP + TN)
  • PPV（精確率）：TP / (TP + FP)

💡 公平性指標的權衡：

  • 不可能三角：無法同時滿足所有公平性定義
  • 準確度 vs 公平性：追求公平可能降低整體準確度
  • 群體間權衡：改善一個群體可能損害另一個

🔍 選擇公平性指標的考量：

  • 應用情境：錯誤類型的相對成本
  • 利害關係人：誰會受到影響
  • 法律要求：是否有明確規範
  • 社會價值：組織的倫理立場

🎯 對抗式去偏的架構：

  • 對抗器（Adversary）：嘗試從預測器的輸出預測敏感屬性 A
  • 訓練目標：最大化預測器準確度，同時最小化對抗器準確度

  • 廣告投放：公平展示機會

⭐⭐⭐⭐

🎯 類別重加權的計算：

  • 反比例加權：w_i = n_total / (n_classes × n_i)
  • 平衡加權：w_i = n_max / n_i
  • 自定義權重：根據業務需求調整

💡 不平衡問題的影響：

  • 模型偏向多數類：傾向預測多數群體
  • 少數類召回率低：漏報少數群體
  • 評估指標誤導：準確度高但無意義
  • 公平性問題：對少數群體不公平

📊 評估不平衡資料的指標：

  • F1-Score：精確率與召回率的調和平均
  • AUC-ROC：不受類別分佈影響
  • PR 曲線：精確率-召回率曲線
  • 分群指標：每個群體分別評估

⚠️ 注意事項：

  • 過度加權可能導致過擬合少數群體
  • 需在驗證集上調整權重
  • 結合多種方法效果更好
📝 題目 #37
⭐⭐⭐⭐
T-Closeness（T-接近性）技術主要是為了降低什麼風險？
A
機率分佈差異造成的識別風險（Identification Risk）
B
資料重複風險（Data Duplication Risk）
C
訓練時間過長
D
演算法震盪問題（Algorithm Oscillation）
💡 詳細解析

正確答案：A

T-Closeness 要求每個等價類（Equivalence Class）中敏感屬性的分佈，與整體資料集的分佈接近（距離 ≤ t），防止透過分佈差異推斷個體資訊。

🎯 T-Closeness 的核心概念：

  • 等價類：具有相同準識別符（如年齡、郵遞區號）的記錄群組
  • 敏感屬性：需要保護的資訊（如疾病、收入）
  • 分佈距離：使用 Earth Mover's Distance 或其他距離度量
  • 閾值 t：允許的最大分佈差異

📊 範例說明：

整體資料集疾病分佈：感冒 60%、糖尿病 30%、癌症 10%

某等價類疾病分佈：感冒 10%、糖尿病 20%、癌症 70%

→ 分佈差異大，可推斷該群組高風險，不滿足 T-Closeness

💡 隱私保護模型的演進：

  • K-Anonymity：基礎保護，防止直接識別
  • L-Diversity：增強保護，防止同質性攻擊
  • T-Closeness：最強保護，防止分佈推論攻擊

⚠️ 實務考量：

  • 隱私 vs 效用：保護越強，資料可用性越低
  • 計算成本：T-Closeness 實作複雜度高
  • 參數選擇：k、l、t 需根據應用場景調整

🎯 SMOTE 的運作步驟：

  • 1️⃣ 選擇少數類樣本：隨機選一個少數類樣本 x
  • 2️⃣ 找 k 近鄰：找出 x 的 k 個最近鄰居（通常 k=5）
  • 3️⃣ 隨機選鄰居：從 k 個鄰居中隨機選一個 x_nn
  • 4️⃣ 線性插值：x_new = x + λ × (x_nn - x)，λ ∈ [0,1]
  • 5️⃣ 重複：直到達到目標樣本數

💡 SMOTE 的變體：

  • Borderline-SMOTE：只在邊界區域生成樣本
  • ADASYN：根據密度自適應生成
  • SMOTE-ENN：結合 SMOTE 和 Edited Nearest Neighbors
  • SMOTE-Tomek：移除 Tomek links 清理邊界

⚠️ SMOTE 的限制：

  • 高維詛咒：高維空間中插值可能不合理
  • 類別重疊：可能在重疊區域生成噪音
  • 類別特徵：不適合類別型特徵
  • 過度生成：可能導致過擬合

🔍 實務建議：

  • 結合重加權和 SMOTE
  • 在訓練集上使用，不要在測試集
  • 先標準化特徵再使用 SMOTE
  • 評估時使用分層交叉驗證

A
遮蔽（Masking）
B
泛化（Generalization）
C
知識蒸餾（Knowledge Distillation）
D
分桶（Bucketing）
💡 詳細解析

正確答案：C

知識蒸餾是模型壓縮技術，用於減小模型大小，不是隱私保護技術。

技術類型目的範例
✅ 遮蔽隱私保護隱藏部分資訊A12***6789
✅ 泛化隱私保護降低精確度25歲 → 20-30歲
✅ 分桶隱私保護將連續值離散化收入 → 低/中/高
❌ 知識蒸餾模型壓縮減小模型大小大模型 → 小模型

🎯 常見隱私保護技術總覽：

  • 遮蔽（Masking）：部分隱藏敏感資訊
  • 泛化（Generalization）：降低資料精確度
  • 分桶（Bucketing）：將數值分組
  • 假名化（Pseudonymization）：替換為代號
  • 抑制（Suppression）：完全移除
  • 擾動（Perturbation）：加入雜訊
  • 加密（Encryption）：密碼學保護
  • 差分隱私（Differential Privacy）：數學保證

💡 隱私保護技術的選擇：

  • 資料類型：數值、類別、文字
  • 使用場景：分析、展示、分享
  • 隱私需求：法規要求的保護等級
  • 資料效用：需保留多少可用性
  • 可逆性：是否需要還原

🔒 隱私保護的層次：

  • Level 1：基礎去識別化（遮蔽、假名化）
  • Level 2：統計保護（泛化、分桶）
  • Level 3：形式化隱私（K-匿名、差分隱私）
  • Level 4：密碼學保護（加密、安全多方計算）

⚠️ 隱私 vs 效用的權衡：

隱私保護程度越高，資料的可用性和準確性通常越低，需要在兩者間找到平衡點。
