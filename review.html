L21101
自然語言處理 NLP
BERT · GPT · Transformer · RAG · TF-IDF · 評估指標

核心架構對比

模型架構特點用途
BERTEncoder-only雙向，MLM預訓練分類、NER、問答
GPTDecoder-only單向，自回歸生成文字生成、摘要
T5/BARTEncoder-DecoderSeq2Seq翻譯、摘要
TransformerAttention機制Self-Attention，位置編碼所有NLP基礎
BERT 雙向看全文做分類；GPT 單向往右生成文字

詞向量技術

技術方法關鍵特點
TF-IDF統計頻率TF×IDF，無法捕捉語意，稀疏
Word2VecCBOW / Skip-gram稠密向量，捕捉語意關係
GloVe共現矩陣分解結合全局統計
BERT Embedding上下文感知同字不同意境→不同向量
TF-IDF 不能理解「語意相似」，只看字頻；Word2Vec 無法區分「蘋果（水果）」vs「蘋果（公司）」

RAG（檢索增強生成）

流程：使用者問題 → Embedding → 向量資料庫相似度搜尋 → 取回相關文件 → LLM 根據文件生成回答
RAG 解決 LLM「知識截止日期」與「幻覺」問題，不需重新訓練模型
RAG 不是微調（Fine-tuning），Fine-tuning 是更新模型權重，RAG 是查外部資料庫

評估指標

指標用途公式/說明
BLEU機器翻譯衡量生成句子與參考句子的 n-gram 重疊率（精確率方向）
ROUGE摘要評估召回率方向，ROUGE-N / ROUGE-L
Perplexity語言模型好壞越低越好，代表模型越「不困惑」
F1-scoreNER / 分類Precision 與 Recall 調和平均
BLEU 看翻譯精確度；ROUGE 看摘要完整度（召回率）

Attention 機制

Self-Attention：每個詞與序列中所有詞計算相關性權重（Q×K/√d → Softmax → ×V）
Multi-Head Attention：多個注意力頭並行，捕捉不同角度的語意關係

提示工程（Prompting）

策略說明
Zero-shot不給例子，直接問
Few-shot給幾個範例再問
Chain-of-Thought (CoT)引導模型「一步一步想」，提升推理準確度
System Prompt設定角色與行為約束
L21102
電腦視覺 CV
CNN · YOLO · ViT · 偵測指標 · 分割類型

核心模型對比

模型類型特點
CNN基礎架構卷積提取局部特徵，池化降維
ResNet分類殘差連接（Skip Connection），解決梯度消失
R-CNN系列偵測（兩階段）先提候選區域，再分類；較慢但準
YOLO偵測（單階段）一次完成，速度快，適合即時
ViT分類（Transformer）把圖切成 Patch，用 Self-Attention
CLIP多模態圖文對齊，雙編碼器
SAM分割Segment Anything，零樣本分割

偵測評估指標

指標說明
IoU（交並比）預測框 ∩ 真實框 / 預測框 ∪ 真實框，通常 >0.5 算偵測成功
Precision預測為正中，真正正確的比例
Recall所有真實正例中，被成功抓到的比例
AP（Average Precision）Precision-Recall 曲線下面積（針對單類別）
mAP所有類別 AP 的平均值，偵測任務的主要指標
mAP 是物件偵測最重要的指標；IoU 門檻通常設 0.5（mAP@0.5）

分割類型

類型細節
語意分割每個像素分類，不區分同類別不同個體
實例分割每個像素分類 + 區分同類別不同個體（Mask R-CNN）
全景分割語意 + 實例的結合
語意=像素分類；實例=像素分類+個體區分；全景=兩者都要

NMS（非極大值抑制）

物件偵測後會有多個重疊的候選框 → NMS 保留得分最高的框，抑制 IoU 超過門檻的重疊框
NMS 不是去掉信心分數低的，而是去掉「跟最高分框重疊過多的」其他框

卷積操作關鍵參數

參數說明
Kernel Size卷積核大小（如 3×3）
Stride步長，越大輸出越小
Padding邊緣補零，保持輸出尺寸
Channel特徵圖數量（深度方向）
L21103
生成式 AI
GAN · VAE · Diffusion · LLM · HHH 原則

三大生成模型對比

模型機制優點缺點
GAN生成器 vs 判別器對抗生成圖像逼真訓練不穩定，模式崩潰
VAE編碼→潛在空間→解碼潛在空間連續、可插值圖像較模糊
Diffusion加雜訊→去雜訊圖像品質最高推論速度慢（多步）
圖像品質：Diffusion > GAN > VAE；訓練難度：GAN最難（容易崩潰）

GAN 深入

生成器（Generator）：從隨機雜訊生成假樣本，目標是騙過判別器
判別器（Discriminator）：判斷樣本是真是假
模式崩潰（Mode Collapse）：生成器只生成幾種固定樣本，失去多樣性
GAN 訓練需要生成器與判別器保持平衡；若判別器太強，生成器梯度消失無法學習

LLM 訓練流程

1. 預訓練（Pre-training）：大量無標注文本，學語言規律
2. 指令微調（Instruction Fine-tuning / SFT）：人工標注的問答對
3. RLHF（人類回饋強化學習）：獎勵模型 + PPO，讓輸出更符合人類偏好

HHH 原則（AI 對齊）

原則說明
Helpful（有用）能有效幫助使用者完成任務
Harmless（無害）不輸出有害、危險的內容
Honest（誠實）不捏造事實，不確定時承認不知道

微調策略對比

方法說明適用場景
Full Fine-tuning更新所有參數資料充足、算力足
LoRA低秩矩陣適配，只更新少量參數低成本微調大模型
Prompt Tuning只訓練提示詞向量最省資源
RAG不改模型，補充外部知識知識更新頻繁
LoRA = Low-Rank Adaptation，凍結原模型，只訓練低秩分解矩陣
L21104
多模態 AI
CLIP · ASR→NLU→TTS · 感測器融合 · Early/Late Fusion

語音 AI 流程

ASR（自動語音識別）→ NLU（自然語言理解）→ TTS（文字轉語音）
語音輸入 → 文字 → 意圖理解 → 文字回應 → 語音輸出
語音助理三步：聽（ASR）→ 懂（NLU）→ 說（TTS）

CLIP 架構

雙編碼器：圖像編碼器（ViT）+ 文字編碼器（Transformer）
對比學習：讓配對的圖文在向量空間中距離近，不配對的距離遠
應用：零樣本圖像分類、圖文搜尋、圖像生成引導

融合策略

融合類型時機優缺點
早期融合（Early Fusion）特徵提取前合併原始資料保留原始互動，計算量大
晚期融合（Late Fusion）各模態分別處理後合併結果靈活，但損失跨模態互動
中期融合（Mid Fusion）在中間特徵層融合折衷方案

感測器融合應用

自駕車：Camera（顏色/紋理）+ LiDAR（距離/深度）+ Radar（速度）
醫療：文字病歷 + CT 影像 + 血液數值
智慧製造：影像 + 振動感測 + 溫度感測

AI 導入評估
ROI · NPV · 回收期 · As-Is/To-Be/Gap · POC

財務評估指標

指標公式判斷標準
ROI（投資報酬率）（效益 - 成本）/ 成本 × 100%>0 代表有利潤
NPV（淨現值）未來現金流折現總和 - 初始投資>0 代表值得投資
回收期（Payback Period）累積現金流 = 初始投資的時間點越短越好
IRR（內部報酬率）NPV = 0 時的折現率>資金成本則投資
NPV 考慮資金時間價值；ROI 不考慮時間價值。若題目問「考慮時間價值」，選 NPV

As-Is / To-Be / Gap 分析

As-Is：現況評估（現在怎麼做？有什麼問題？）
To-Be：目標藍圖（AI 導入後應該變成什麼？）
Gap：差距分析（需要做哪些改變？）
POC（概念驗證）：在正式上線前，用小規模實驗驗證可行性

KPI 設計原則

SMART 原則：Specific（具體）、Measurable（可量測）、Achievable（可達成）、Relevant（相關）、Time-bound（有時限）

導入成熟度評估

層面評估重點
技術成熟度技術是否夠成熟、有無替代方案
資料準備度資料量、品質、標注完整性
組織能力人才、文化、流程配合度
合規風險法規、隱私、倫理考量

層級例子規範

OECD AI 原則（五大）

1. 包容性成長：AI 應促進人類福祉
2. 以人為本：尊重法治、人權
3. 透明性：AI 決策應可解釋

模型漂移類型

類型定義嚴重度處置
資料漂移（Data Drift）輸入特徵 X 分佈改變（P(X)變）中重採樣、微調

L21301
數據準備與模型選擇
缺值處理 · 異常值 · 特徵編碼 · 縮放 · PCA

缺失值三類型

類型全名說明處理
MCAR完全隨機缺失缺失與任何資料無關直接刪除、均值填補
MAR隨機缺失缺失與其他已觀測欄位有關多重插補（MI）
MNAR非隨機缺失缺失與自身值有關（如高收入不申報）最難處理，需專業判斷

異常值偵測方法

方法規則適用
Z-score|Z| > 3 視為異常常態分佈資料
IQR<Q1-1.5×IQR 或 >Q3+1.5×IQR不需常態假設，穩健
Isolation Forest基於隔離樹，異常點更容易被孤立高維度、大數據
LOF局部離群因子，基於密度非規則形狀分佈

特徵編碼

方法適用注意
Label Encoding有序類別（如：小/中/大）數字大小有意義
One-Hot Encoding無序類別（如：城市名）維度爆炸問題
Target Encoding高基數類別需防止資料洩漏（target leakage）

特徵縮放

方法公式特點
Min-Max 正規化(x - min) / (max - min)縮放到 [0,1]，對異常值敏感
Z-score 標準化(x - μ) / σ均值0標準差1，假設常態分佈
Robust Scaler(x - Q2) / IQR對異常值穩健，用中位數和IQR
有異常值 → Robust Scaler；需要 [0,1] → Min-Max；一般情況 → Z-score

PCA（主成分分析）

目的：降維，保留最大變異方向
步驟：標準化 → 計算共變異矩陣 → 求特徵值/向量 → 選前 k 個主成分
缺點：主成分無法直接解釋為原始特徵（可解釋性差）
PCA 是「降維」不是「特徵選擇」！降維後的主成分是線性組合，非原始特徵
L21302
AI 系統集成與部署
藍綠/金絲雀/A-B · K8s · Docker · SLA/SLO/SLI

部署策略比較

策略方式特點
藍綠部署兩套環境，一次切換全部流量回滾快，成本高（兩套環境）
金絲雀釋出先導少量流量（如5%），逐步擴大風險低，可觀察新版表現
A/B Testing同時兩版本，比較指標實驗導向，有統計假設
滾動更新逐台替換，不停機K8s 預設，平滑過渡

Kubernetes (K8s) 重要概念

概念說明
Pod最小部署單位，包含一個或多個容器
HPA（水平自動擴展）根據 CPU/記憶體自動增減 Pod 數量
VPA（垂直自動擴展）自動調整單一 Pod 的資源配額
Liveness Probe偵測容器是否還活著，死了就重啟
Readiness Probe偵測容器是否準備好接受流量
Liveness = 還活嗎？（不活就重啟）；Readiness = 準備好了嗎？（沒準備好就不導流量）

SLA / SLO / SLI

縮寫全名說明
SLA服務等級協議對外的契約（如：99.9% 可用性）
SLO服務等級目標內部目標（通常比 SLA 嚴格）
SLI服務等級指標實際量測的指標（如：延遲、錯誤率）

MLflow 用途

實驗追蹤：記錄超參數、指標、模型版本
模型倉庫：版本管理、模型上線/退休狀態管理
套件打包：MLproject 格式，確保可重現
L22101
敘述性統計
集中趨勢 · 離散程度 · 偏態 · 峰態

集中趨勢三均值

均值公式適用
算術平均數Σx / n一般情況（對異常值敏感）
幾何平均數(x₁×x₂×…×xₙ)^(1/n)成長率、比率型資料
調和平均數n / Σ(1/xᵢ)速率（如：平均速度、F1-score）
F1-score 是 Precision 與 Recall 的調和平均（Harmonic Mean）
有極端值時，中位數比算術平均數更能代表「典型值」

離散程度指標

指標說明
變異數（Variance）偏差平方的平均，單位是原單位的平方
標準差（SD）變異數的平方根，與原資料同單位
IQR（四分位距）Q3 - Q1，不受極端值影響
變異係數（CV）SD / 平均數，跨單位比較離散程度

偏態與峰態

指標值含義
偏態（Skewness）= 0對稱分佈
> 0（右偏/正偏）尾巴在右邊，平均數 > 中位數 > 眾數
< 0（左偏/負偏）尾巴在左邊，平均數 < 中位數 < 眾數
峰態（Kurtosis）= 3（excess=0）常態分佈（Mesokurtic）
> 3（excess>0）尖峰（Leptokurtic），尾巴厚
< 3（excess<0）平峰（Platykurtic），尾巴薄
右偏：長尾巴在右；平均數被右邊大值拉高 → 平均 > 中位 > 眾

機率分佈
離散 vs 連續 · 常態 · 二項 · 卜瓦松 · 指數

分佈速查表

分佈類型參數適用情境
伯努利（Bernoulli）離散p單次二元實驗（成功/失敗）
二項（Binomial）離散n, pn 次獨立伯努利試驗的成功次數
卜瓦松（Poisson）離散λ單位時間/空間內的事件次數
常態（Normal）連續μ, σ自然現象，中央極限定理
指數（Exponential）連續λ等待時間（卜瓦松事件間隔）
卡方（Chi-square）連續k（自由度）假設檢定（獨立性、適合度）
t 分佈連續ν（自由度）小樣本均值推論
F 分佈連續d1, d2ANOVA、變異數比較

常態分佈關鍵數字

68-95-99.7 法則
μ ± 1σ：68% 資料
μ ± 2σ：95% 資料
μ ± 3σ：99.7% 資料
中央極限定理：不管原分佈如何，樣本量夠大（n≥30）時，樣本均值趨向常態分佈

卜瓦松 vs 指數

卜瓦松：「1 小時內到幾個客人？」（次數）λ = 平均到達率
指數：「下一個客人要等多久？」（時間）1/λ = 平均等待時間
兩者互為相關，同一個 λ

假設檢定與統計推論
p 值 · α · 型一/二錯誤 · t-test · ANOVA · 卡方

假設檢定基本框架

H₀（虛無假設）：假設沒有差異/效果（保守立場）
H₁（對立假設）：假設有差異/效果（研究者想證明的）
p 值：假設 H₀ 為真，觀察到目前資料（或更極端）的機率
決策：p < α（通常 0.05）→ 拒絕 H₀

兩類錯誤

錯誤說明代價對應
型一錯誤（α）H₀ 是真的，卻拒絕（False Positive）誤報警顯著水準 α
型二錯誤（β）H₀ 是假的，卻接受（False Negative）漏掉效果1-β = 統計檢定力
型一 = 冤枉好人；型二 = 放走壞人

統計檢定選用指引

情境檢定方法
一個樣本均值與已知值比較單樣本 t 檢定
兩個獨立組均值比較獨立樣本 t 檢定
同一組前後測比較配對樣本 t 檢定
三組以上均值比較單因子 ANOVA
兩個類別變數的獨立性卡方獨立性檢定
觀察值與期望分佈的吻合度卡方適合度檢定
不滿足常態假設的兩組比較Mann-Whitney U 檢定（非參數）

ANOVA 後的多重比較

ANOVA 只告訴你「至少有一對組別有差異」，不告訴你是哪對
→ 需要事後檢定（Post-hoc Test）：Tukey HSD、Bonferroni 校正
多重比較問題：做越多檢定，型一錯誤累積越高（α 膨脹）→ 需要校正

大數據的統計陷阱

p 值膨脹問題：樣本量越大，p 值越容易小於 0.05（統計顯著）
→ 即使效果微小，大數據中也容易達到統計顯著
→ 統計顯著 ≠ 實務顯著！
大數據中要同時報告：p 值 + 效應量（Effect Size）

指標適用判斷

幻覺（Hallucination）

定義：模型編造不存在的事實、假論文、假法條
危險：文法正確、語氣篤定，但內容是假的
原因：模型是在「猜下一個高機率詞」，不是在查資料庫
解法說明
RAG先從外部知識庫檢索，再根據檢索結果回答
提示工程要求引用來源，告知「不知道就說不知道」
RLHF人類標注者對回答打分，優化模型輸出

合成資料（Synthetic Data）

三大使用情境：
1. 樣本不足/不平衡（如：詐欺案例太少）
2. 罕見情境模擬（如：自駕車邊緣案例）
3. 隱私保護（如：醫療病歷不能直接用）
合成資料仍需評估「成員推論攻擊」風險，無法百分百保護隱私

偏誤 vs 幻覺辨別

偏誤：有偏見的真實，學到不公平傾向 → 關鍵詞：刻板印象、歧視
幻覺：一本正經的胡說八道，捏造事實 → 關鍵詞：捏造、假論文、假法條

資料識別等級

類型特徵例子
PII（直接識別）單一欄位即可找到你姓名、身分證、Email、IP
準識別資訊多欄位組合才能鎖定生日+性別+郵遞區號
敏感個人資料外洩會造成實質傷害病歷、基因、財務、宗教

基礎去識別技術

技術做法考試關鍵字
遮蔽（Masking）部分字元打碼（王ＯＯ）UI 防窺，最基礎
泛化（Generalization）精度降低（1995→1990年代）「將精度降低」必選此
分桶（Bucketing）數值裝入區間（34歲→30-39歲）「數值裝進區間」
雜湊（Hashing）單向轉換，不可逆注意彩虹表攻擊，需加鹽
隨機擾動加入隨機噪音差分隱私的實現手段

四大進階 PETs 技術

技術核心機制適用情境口訣
K-匿名每筆資料至少與 K-1 人相同（泛化+抑制）發布公開資料集藏木於林
差分隱私加噪音 + ε 隱私預算發布統計報表、遙測加了沙子的統計
聯邦學習資料不出本地，只傳模型參數跨院訓練、手機鍵盤資料不動模型動
同態加密直接在密文上運算雲端算但不看明文蒙眼算數學

PETs 情境快配

「發布一萬筆明細供外界分析」→ K-匿名
「發布平均薪資報表，防個體被反推」→ 差分隱私
「跨醫院訓練 AI，病歷不得出院」→ 聯邦學習
「借雲端算力，但雲端不能看明文」→ 同態加密

K-匿名的弱點與補強

弱點說明補強方案
同質性攻擊K 個人敏感屬性完全相同，仍可推論L-多樣性：敏感屬性至少 L 種
背景知識攻擊用外部知識排除可能選項T-接近性：分布接近整體

差分隱私關鍵

ε（隱私預算）：越小 → 噪音越大 → 隱私保護越強 → 準確度越低
拉普拉斯機制：經典噪音方式
高斯機制：深度學習訓練（DP-SGD）常用
聯邦學習並非絕對安全！梯度也可能洩漏隱私（梯度洩漏攻擊）

GDPR 六大原則

#原則違反的典型考題
1合法、公平與透明未告知就收集
2目的限制說好做A，偷偷拿去做B
3資料最小化「先全收，以後可能用到」
4正確性資料錯了不更正
5保存期限限制用完不刪，無限囤積
6完整性與機密性沒有加密保護
GDPR 最高罰則：2000 萬歐元 或 全球年營業額 4%，取其高

被遺忘權與 Model Card

被遺忘權（Right to be Forgotten）：使用者可要求刪除其個資，引發「機器遺忘」問題
Model Card：AI 模型的透明度說明書，記載訓練資料、偏誤、適用範圍
最小權限原則（RBAC）：只給完成當前工作所需最低權限
